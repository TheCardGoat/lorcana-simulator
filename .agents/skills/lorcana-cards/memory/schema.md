# Memory Schema

Keep this memory small and operational.

Do not append long execution logs here.

## Update Rules

- Prefer editing the summary sections in `bank.md` in place.
- Add a dated note only when a new learning changes future card work.
- Keep each dated note short and durable.
- Delete stale notes instead of preserving history for its own sake.

## Allowed Summary Sections

- `Current State`
- `Guardrails`
- `Trusted Patterns`
- `Known Gaps`
- `Do Not Trust`
- `Recent Learnings`

## Dated Note Template

```markdown
## YYYY-MM-DD - <topic>

- signal: <what changed or was learned>
- impact: <how future work should change>
- verification: <command or evidence>
```
